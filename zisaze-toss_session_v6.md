### 3. 디버깅 기록 — "검토 요청하기" 버튼 비활성 원인 규명 (재발 방지용)
- 증상: 폼을 다 채워도 버튼 disabled. DOM 조작으로 속성을 제거해도 클릭 무반응(내부 핸들러 `P?.()` 가드).
- **원인: 출시 노트 숨은 maxLength 검증.** 처음 넣은 1,220자 노트가 한도 초과 → react-hook-form `errors.releaseNotes.type === "maxLength"` (에러 메시지는 빈 문자열이라 화면에 안 보임!) → isValid=false → 버튼 disabled 유지. **792자로 줄이니 즉시 활성화·제출 성공.** (한도 정확값 미확인 — 1,000자 추정. 800자 이하 안전.)
- 진단법: 버튼 fiber를 올라가 `InitialVersionReviewModal` 컴포넌트 훅에서 RHF formState(`{isValid, errors}`)를 직접 읽음. DOM의 disabled 속성이 아니라 **React memoizedProps의 disabled가 진실**(속성만 지우면 활성처럼 보이지만 무반응).
- 입력 팁: 콘솔 폼은 controlled. MCP `fill`은 기존 값에 **이어붙임**(교체 아님) — native setter + `_valueTracker.setValue('')` 리셋 + input 이벤트 디스패치로 교체해야 React 상태에 반영.
- 영어 기능 이름 검증: 개발자센터 문서는 "첫글자만 대문자"라지만 **실제 콘솔 검증은 "각 단어의 첫 글자는 대문자 또는 숫자"**(Title Case). 문서보다 콘솔 실검증이 정답.
