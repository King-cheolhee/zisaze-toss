# P3-3 실기기 테스트 프로토콜 v1 (사용자 실행용)

> 작성: 2026-07-06. 실행: [사용자]. 소요 약 20~30분. 준비물: **Android 폰 1대 + iPhone 1대(같은 토스 계정 로그인)**, 콘솔 샌드박스 QR(P3-2에서 번들 업로드 후 발급).
> 핵심 목적: **R1 게이트 — getAnonymousKey가 재실행·재설치·기기 간에 같은 값으로 유지되는지**(공식 문서에 미명시라 실기기로만 확인 가능). 실패하면 북마크·선호가 초기화되는 치명 문제라 최우선.

## 판정 방법 (개발 도구 없이 화면으로)

hash는 화면에 안 보이므로 **"북마크·선호가 유지되면 같은 사용자(=같은 hash)"** 로 판정한다.
각 단계의 결과를 ✅/❌로 기록하고, ❌가 나오면 그 단계 번호와 화면 상태(스크린샷)를 남긴다.

## ⚠️ 버그로 오인하지 말 것 (알려진 동작)

1. **"전부 건너뛴" 사용자에게 온보딩이 한 번 더 뜰 수 있어요** — 온보딩을 모두 건너뛰어 서버에 아무 정보가 없는 상태에서, 앱 저장소가 초기화된 채 재진입하면(재설치 등) 보안상 온보딩이 다시 나옵니다. 데이터 손실은 없고 다시 건너뛰면 됩니다(의도된 동작 — 세션 v3 "알려진 한계").
2. 샌드박스(QR 테스트)의 익명키는 **mock일 수 있음** — 콘솔 안내에 따라 C·D 판정은 실기기+실배포 환경 기준이 정확합니다. 샌드박스에서 C1이 애매하면 기록만 하고 검토 요청 전 재확인.
