## 4. 스킬·지식 결합 방식

- **주입(스타일)**: ponytail 철학 + recall 교훈 브리핑 + (필요시) 라이브러리 페르소나(`_HOUSE-RULES.md` 선행 → 보안·접근성·프론트)를 코딩/리뷰 프롬프트에 넣는다.
- **게이트(diff 검사)**: `ponytail-review`·`code-review`·`cso`·`recodex`는 오케스트레이터가 Phase 끝에 메인 루프에서 실행(fan-out 서브에이전트 안에 넣지 않는다).
- **세컨드브레인**: 오케스트레이터가 대표로 recall(시작)·lesson(종료). *주의: claude.ai 커넥터라 IDE 세션에선 되지만 cron/headless면 빠질 수 있음.*
- *참고: fan-out 서브에이전트가 Skill 도구를 직접 호출하는 것은 보장되지 않음 → 위 "주입/게이트"로 우회하는 것이 정석.*
